Delete a config container

Usage:
  config delete KEY [options]
  config delete --help | -h

Aliases:
  del

Parameters:
  KEY
    Dot notation string describing the intended config container

Options:
  --file, -f PATH
    Path to the config file (default: $CONFIG_FILE)

  --on FIELD:VALUE
    Select a record by FIELD:VALUE. May be repeated.

  --if-empty
    Only delete when the container has no values

  --if-exists
    Do nothing when KEY is not set

  --dry, -n
    Print the updated config without modifying the file

  --diff, -d
    Print a unified diff without modifying the file

  --color, -c
    Colorize diff output

Examples:
  # Delete a container
  config delete -f config.toml style

  # Use a default file for repeated commands
  export CONFIG_FILE=config.toml
  config delete servers.1

  # Use the short command alias
  config del servers.1

  # Delete a matching array record
  config delete servers --on name:web

  # Delete only when the container is empty
  config delete tui.keymap.composer --if-empty

  # Delete if the container exists
  config delete style --if-exists
