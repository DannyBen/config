Topic: formats

The `config` command supports TOML, YAML, JSON, and INI files.

**Extensions**

  TOML
    `.toml`

  YAML
    `.yaml`, `.yml`

  JSON
    `.json`

  INI
    `.ini`

**Unknown extensions**

  Files with unknown extensions are detected from their contents. TOML and INI
  are checked first because their syntax can overlap. If both match, `config`
  refuses the file and asks for a format hint.

  Add a format hint as the first non-empty line when needed:

    `# format: toml`
    `# format: ini`
    `# format: yaml`

  JSON does not support format hints because JSON files cannot contain comments.

**Write behavior**

  TOML and YAML
    Edits are source-preserving where possible. Comments, nearby formatting,
    table style, YAML anchors, and YAML array style are preserved when the edit
    can be made safely.

  JSON
    Edits rewrite the whole document as canonical pretty JSON with two-space
    indentation and a trailing newline. Standard JSON is supported; comments are
    not allowed.

  INI
    INI files are modeled as global keys and named sections. `get`, `list`,
    `dump`, `set`, `unset`, and section `delete` are supported. Array
    operations and record selectors do not apply to INI files.

    Full-line comments starting with `#` or `;` are ignored. Inline comments
    are not parsed, so `port = 3000 # dev` has the value `3000 # dev`.

**Paths**

  Dot paths address nested config data. Use `..` for a literal dot inside a key
  segment:

    `network.public..port` -> key `public.port` under `network`

**INI files**

  INI paths support global keys and one section level:

    `key`
    `section.key`

  The same `..` escaping works for literal dots in INI section or key names:

    `server..port`      -> key `server.port`
    `env..prod.server`  -> section `env.prod`, key `server`
    `env.prod..server`  -> section `env`, key `prod.server`

  More than one unescaped separator dot is not supported.

**YAML arrays**

  New YAML arrays are written in block style. Existing block arrays stay block
  style. Existing flow arrays stay flow style until they grow to five values.
